做 EdgeEver，不是因为世界上缺一个笔记应用。

恰恰相反，笔记应用太多了。轻量的、重型的、协作优先的、Markdown 优先的、白板式的、数据库式的，每一种都有自己的拥护者。但长期使用笔记软件的人心里大概都知道：真正难替代的并不是某个功能，而是一套已经长进肌肉记忆里的工作流。

对我来说，这套工作流很接近经典印象笔记：左边是笔记本树，中间是笔记列表，右边是正在写的内容。打开、搜索、归类、移动、继续编辑，所有动作都应该足够直接。它不需要每一步都显得很酷，它需要稳定、清楚、快，并且让我放心把多年积累的内容放进去。

但这几年，我越来越难从现有产品里得到这种放心。

---

### 我想要的是一个可靠的个人知识库

很多人并不是要一个越来越庞大的效率平台。我们只是想要一个可靠的个人知识库：

- 能快速记录和整理日常资料
- 能按熟悉的笔记本层级管理内容
- 能保存图片、附件、富文本和长文
- 能在电脑和手机上都顺手使用
- 能在需要时把数据拿出来

这些需求并不花哨，但它们很基础。基础能力一旦不稳定，笔记软件就会从工具变成负担。

我曾经喜欢印象笔记的原因，也正是因为它在很长一段时间里把这些基础能力做得足够好。三栏布局、笔记本、搜索、跨设备同步，这些东西构成了一种朴素但高效的个人资料管理方式。

问题是，随着产品越来越复杂，我想要的那部分体验反而被包在了越来越厚的外壳里。

---

### 臃肿、封闭和迁移焦虑

对个人用户来说，笔记软件最可怕的事情不是少一个按钮，而是数据被困住。

当一个产品变得越来越重，商业化和附加功能越来越多，性能、内存占用和日常响应速度开始让人分心时，你自然会开始想：如果有一天我想离开，我能不能体面地把自己的内容带走？

这个问题并不总是有舒服的答案。

笔记里通常有多年资料、图片、附件、剪藏、碎片想法和未完成的计划。它们不是一批普通文件，而是一个人的知识轨迹。迁移如果依赖不稳定的第三方插件，或者只能导出到不够开放、不够容易二次处理的格式，用户就会一直处在一种隐形的焦虑里。

更进一步，AI Agent 正在成为新的工作入口。今天我们已经可以让 Codex、Claude Code、Cursor、Antigravity 这类工具帮我们写代码、整理资料、处理重复任务。那笔记系统也应该能被这些工具安全、清晰地访问。

如果我的知识库不能通过 API 或 MCP 被授权读取、整理、归档、打标签，它就很难进入下一代工作流。

---

### 轻量替代品很好，但我仍然想要三栏体验

我也看过一些更轻量、更开放的笔记产品。它们通常很干净，也更接近开发者喜欢的形态。

但对我来说，还有一个缺口：经典印象笔记式的三栏体验依然很重要。

三栏不是怀旧。它是一种高效的信息密度：

- 左侧让我知道内容放在哪里
- 中间让我快速扫过一组笔记
- 右侧让我立即进入编辑状态

这种布局尤其适合长期积累的资料库。它不强迫我先进入某个复杂空间，也不要求我为了管理内容先设计一套系统。它更像一个工作台：打开就能用，东西都在手边。

所以 EdgeEver 一开始就不是想做一个“全新范式”的笔记软件。它想填补的是一个更具体的空白：保留熟悉、直接的三栏笔记体验，同时把底层数据、部署方式和 Agent 能力打开。

---

### 为什么选择 Cloudflare-native 自托管

我希望 EdgeEver 是普通个人也能长期使用的自托管产品。

传统自托管常常意味着服务器、数据库、对象存储、备份、运维和账单。对很多人来说，这些门槛会把“拥有自己的数据”变成一句好听但很难坚持的话。

Cloudflare 的 D1、R2、Workers 和 Pages 给了一个很适合个人知识库的组合：

- D1 存储结构化数据和笔记内容
- R2 存储图片与附件
- Workers 提供 API
- 静态前端可以部署在边缘网络上

在个人使用规模下，这套组合有机会做到接近零成本。更重要的是，它不需要用户长期维护一台服务器。你 Fork 仓库、配置资源、部署，之后同步上游更新再重新部署，就能持续使用。

这就是 EdgeEver 想表达的一个价值：自托管不应该只属于愿意长期运维服务器的人。

---

### 开放数据是底线，不是卖点

EdgeEver 的内容模型从一开始就尽量保持清楚：

```text
content_json      TipTap/ProseMirror 文档，编辑器权威格式
content_markdown  API、Agent、导入导出使用
content_text      搜索、摘要和索引使用
```

这样做并不是为了显得技术上复杂，而是为了避免把所有场景都塞进一种格式里。

编辑器需要结构化文档，API 和迁移需要 Markdown，搜索需要纯文本。每一种形态都有自己的用途。数据越清楚，未来迁移、备份、自动化和二次开发就越轻松。

我希望用户使用 EdgeEver 时，有一种非常明确的安全感：这些内容是你的。你可以通过 REST API 读取它，可以通过 MCP 交给 Agent 整理它，也可以在需要时把它迁移出去。

---

### MCP 让笔记重新变成可行动的知识

AI Agent 友好是 EdgeEver 很重要的一部分。

过去，笔记软件更多是在帮我们“保存信息”。但现在，仅仅保存已经不够了。我们还希望工具能帮我们：

- 总结一批随机记录的想法
- 给旧笔记自动补标签
- 从长期记录里整理项目脉络
- 根据笔记内容生成待办、文档或方案
- 在迁移资料时自动清洗格式和层级

这些事情如果只能靠复制粘贴，就很快会变得笨重。MCP 的意义在于，用户可以明确授权 AI Agent 访问自己的 EdgeEver 实例，让它在边界清楚的前提下读取和整理笔记。

这也是我想做 EdgeEver 的原因之一：笔记不应该只是沉睡的资料仓库，它应该能进入自动化工作流，变成可以被调用、被组织、被重新使用的知识。

---

### EdgeEver 想成为什么

EdgeEver 不想成为一个包办一切的超级应用。

它更像一个开放、轻量、可长期拥有的个人笔记底座：

- 保留经典三栏体验
- 支持富文本、图片、嵌套笔记本和历史版本
- 可以部署在 Cloudflare 上
- 数据模型清晰，支持 REST API、OpenAPI 和 MCP
- 对个人用户尽量低成本
- 对 AI Agent 足够友好

如果你怀念印象笔记早期那种直接、稳定、可依赖的工作台，又希望今天的笔记系统能够开放、自托管、能接入 AI Agent，那么 EdgeEver 就是在为这个交叉点而做。

我做它，是因为我想要这样一个工具：熟悉但不封闭，轻量但不简陋，个人可拥有，又能面向未来的自动化工作流。

这就是 EdgeEver 的起点。
